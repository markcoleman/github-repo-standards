Quickstart

Adopt the required workflow.

Add this workflow to a repository that should be validated. Replace OWNER with the account or organization that owns this standards repository.

.github/workflows/required-repo-standards.yml 
name: Required Repository Standards

on:
  pull_request:
  push:
    branches:
      - main

permissions:
  contents: read
  issues: write
  pull-requests: write

jobs:
  repo-standards:
    uses: OWNER/github-repo-standards/.github/workflows/repo-standards.yml@main

Default standards

Small rules, high leverage.

Check 010

Root README required

Validates a root-level README file and enforces configurable minimum content through REQUIRED_README_PATH and MIN_README_BYTES.

Read implementation notes

Check 020

CODEOWNERS required

Accepts CODEOWNERS or .github/CODEOWNERS when at least one non-comment entry includes an owner.

Checks 030-060

Architecture

How a standards run moves through the system.

The reusable workflow checks out the caller repository, checks out this standards repository at the workflow ref, runs the composite action, and optionally updates a single PR comment.

Extend

Add repository-specific checks without forking the core.

Place focused shell checks in the caller repository and pass their directory to the reusable workflow. Bundled checks run first.

Workflow input 
jobs:
  repo-standards:
    uses: OWNER/github-repo-standards/.github/workflows/repo-standards.yml@main
    with:
      check-directory: .github/repo-standards/checks
Check helper contract 
#!/usr/bin/env bash
set -euo pipefail

if [[ -f "LICENSE" ]]; then
  pass "License is present" "LICENSE exists at the repository root."
else
  fail "License is present" "Add a root-level LICENSE file."
fi

Security and supply chain

Use GitHub-native controls as a baseline.

Pair the standards workflow with Dependabot, CodeQL, OpenSSF Scorecard, least-privilege workflow permissions, SARIF upload, secret scanning, push protection, and branch rulesets.

.github/dependabot.yml 
version: 2
updates:
  - package-ecosystem: "github-actions"
    directory: "/"
    schedule:
      interval: "weekly"

Recommended controls

  • Require repository standards and security analysis checks before merge.
  • Enable Dependabot alerts, security updates, secret scanning, and push protection.
  • Review third-party actions and keep workflow permissions scoped to each job.
  • Upload SARIF from supported analyzers so findings appear in code scanning.

Team ownership

Combine CODEOWNERS with catalog metadata.

CODEOWNERS routes review requests. Repository ownership metadata and a Backstage-compatible catalog descriptor add operational context for teams, service catalogs, escalation, and developer portals.

catalog-info.yaml 
apiVersion: backstage.io/v1alpha1
kind: Component
metadata:
  name: github-repo-standards
spec:
  type: service
  lifecycle: production
  owner: platform-engineering
  system: developer-experience

Operate

Designed for required checks and review flow.

A hidden marker lets later runs update the same standards comment instead of creating repeated noise.

Local parity
